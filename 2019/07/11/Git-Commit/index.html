优雅地Git Commit

规范化以增加可读性

Word count: 325Reading time: 1 min

在公司实习经常commit，于是学习和记录相关规范，以便日后使用，有利于项目维护。

Git ignore

语法规则这里就不赘述了，主要是记录（提醒）哪些类型的文件应该被忽略（不提交）

  1. /target

  2. 为了适配 git cz 而 创建的 package.json 文件

  3. 其他并非是项目编译运行所必须的文件，比如IR，*.exe，*.class 文件

  4. 可以直接从 .gitignore模版 查看和使用

  5. 强制添加因 .gitignore而加不进的文件，可以使用 -f 参数ingit add.
    Ps： .gitignore本身会加入 CVS, 可以对其进行版本控制。

commit 格式标准

commitizen

一个用来规范化 commit 的命令行工具，github 地址

安装

  1. 安装 NPM，因为项目依赖于 node.js 的包管理工具。
    brew install node
  2. 安装 Commitizen
    npm install -g commitizen
  3. 安装changelog

    使用

  4. 执行如下命令添加 package.json 避免下条命令报错
    npm init --yes
  5. 使其支持Angular的Commit message格式
    commitizen init cz-conventional-changelog --save --save-exact
  6. 添加 changelog.md 文件
    conventional-changelog -p angular -i CHANGELOG.md -s
  7. 以后使用 git cz 代替 git commit，按引导操作即可完成
